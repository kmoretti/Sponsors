# project-dev 触发口令备忘

## 默认主入口
- 用 project-dev 处理这个任务
- 按 project-dev 工作流做
- 先走 project-dev

含义：
- 先读 `history.md`
- 再读 `wiki.md`
- 开始任务
- 改完回写 `history.md`

---

## 项目文档类
- `/wiki`
- 用 project-dev 更新 wiki
- 重新整理项目 wiki

适合：
- 新接手项目
- 项目结构变化后更新文档
- README 更新后同步项目知识

---

## 低 token 模式
- 用 project-dev，尽量省 token
- caveman 风格
- 短一点，直接结论

适合：
- 你已经了解上下文
- 只想快速推进任务
- 不想看长篇解释

---

## Bug 排查模式
- 用 project-dev 排查这个 bug
- 按 diagnose 思路处理
- 先复现再修

适合：
- 报错
- 行为异常
- 性能回退
- 根因不清的问题

---

## 稳妥修复模式
- 按 tdd 思路修
- 先复现/先测试再改
- 先让问题可验证

适合：
- 容易改坏的逻辑
- 回归风险高的模块
- 需要更稳的修改方式

---

## 需求澄清模式
- 先 grill 一下这个需求
- 先 grill-with-docs
- 先别写代码，先帮我把需求问清楚

适合：
- 需求含糊
- 存在多个方案
- 你自己也没完全想清楚

---

## 全局理解模式
- 先 zoom-out
- 先从整体看这个模块
- 先理解结构，再动手

适合：
- 陌生代码区
- 跨模块改动
- 担心只看局部导致误改

---

## 我最推荐的 6 句
1. 用 project-dev 处理这个任务
2. /wiki
3. 用 project-dev，尽量省 token
4. 按 diagnose 思路排查
5. 按 tdd 思路修
6. 先 zoom-out 再改
